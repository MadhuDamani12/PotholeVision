# PotholeVision: Automating Pothole Detection and GeoMapping

## Overview

The road inspection process for cities is highly manual, slow, and expensive. Municipalities desperately need a scalable way to identify road damage faster and prioritize repairs. To solve this, we developed PotholeVision, an automated pothole detection and geo-mapping solution. We built a big data pipeline that ingests video and image data in batches, extracts frames, and evaluates them using a ResNet18-based Convolutional Neural Network. This technical pipeline drives direct business value by pushing positive pothole predictions into a dynamic ArcGIS dashboard. Through this dashboard, Public Works planners can view pothole counts, exact locations, and street traffic volumes to prioritize repairs on the busiest roads. Ultimately, PotholeVision drives faster repairs, better allocation of limited maintenance resources, and a significant reduction in vehicle accidents.

The machine learning component is a strict pothole image classifier. It flags likely pothole frames from road videos, then hands those prediction outputs to the Databricks and ArcGIS workflow for mapping and dashboarding. The project evolved from a simple CNN experiment into a fuller end-to-end solution: model training, external validation, Databricks video/frame processing, and ArcGIS dashboard visualization.

The core model is a strict binary classifier:

- `pothole`
- `non_pothole`

The final deployment candidate is the v3 ResNet18 classifier, which was improved by adding more diverse pothole examples and validating on an independent normal/pothole dataset.

## Pipeline Flow

The project pipeline is designed around these stages:

1. Raw road videos are ingested into Databricks storage.
2. Videos are split into image frames.
3. Each frame is passed into the CNN classifier.
4. The model outputs a pothole probability for each frame.
5. Positive/high-probability frames are written into prediction tables.
6. ArcGIS reads the prediction table and creates map-ready geometry features.
7. The final dashboard visualizes likely pothole locations, probability, and related traffic context.

The CNN model cannot classify a normal Spark feature vector directly. It needs image data as tensors, so the pipeline stores metadata and prediction results in tables while running the CNN on extracted frame images.

## ArcGIS Dashboard Flow

`reports/ArcGIS_Data_Flow.pdf` describes the dashboard handoff:

- ArcGIS connects to the Databricks catalog containing prediction outputs.
- The pipeline reads the prediction table generated by the model.
- Coordinates are joined or generated so ArcGIS can map pothole points.
- Nearby predicted pothole points can be joined together to reduce duplicate markers.
- The output feature layer is refreshed on a schedule.
- The dashboard displays pothole probability through point size/color and can be combined with traffic layers such as AADT.

## Key Findings

- High false-negative rates were mainly caused by domain shift, meaning potholes from new datasets looked visually different from training potholes.
- Alligator cracks are visually close to potholes, so they were better handled as a challenge set instead of a normal negative class.
- Hard negative mining reduced alligator false positives but made the model too conservative when overused.
- V3 improved pothole recall substantially by adding more diverse YOLOv8 pothole samples.
- On the independent Neha normal/pothole benchmark, v3 reached about 90.6% pothole recall and 9.4% miss rate at threshold 0.50.

## Usage: Local Inference

```python
from src.pothole_classifier import PotholeClassifier

clf = PotholeClassifier("./model_artifacts")
result = clf.predict("path/to/road_frame.jpg")
print(result)
```

The classifier accepts common image formats supported by Pillow, such as `.jpg`, `.jpeg`, and `.png`.

## Datasets Used

- **RDD 2020 Road Damage Dataset:** Multi-country road damage dataset used as the main starting point.
- **Sovit Pothole Dataset:** Dashcam-style positive and negative samples used for domain adaptation.
- **YOLOv8 Pothole Segmentation Dataset:** Additional pothole-positive samples used to improve v3 recall.
- **Neha Normal/Pothole Dataset:** Independent external benchmark for final validation.

Raw datasets are not stored in this repository because of size and storage constraints.
